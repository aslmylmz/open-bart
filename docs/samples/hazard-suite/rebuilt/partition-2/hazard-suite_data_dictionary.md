# Data Dictionary — hazard-suite

Generated by the instrument (version 1.2.1) from its scoring models — regenerated automatically whenever the schema changes; do not edit by hand. Together with the frozen study config (`hazard-suite_study.json`) and the provenance record (`hazard-suite_provenance.json`), this makes the output directory self-describing and ready to archive or upload (e.g. to OSF) as-is.

Metrics mode: `advanced` — the full metrics surface (the v1.0.0 default).

## Master CSV columns (`hazard-suite_results.csv`)

One row per completed session.

| Column | Description |
|--------|-------------|
| `timestamp_utc` | Session write time (UTC, filename-safe format) — shared by the session's files and its study-wide CSV rows. |
| `session_id` | The client-generated session identifier (UUID). |
| `candidate_id` | The participant ID entered at the start of the run. |
| `average_pumps_adjusted` | Mean pumps per non-exploded balloon (adjusted BART score; Lejuez et al., 2002). None (null JSON / empty CSV cell) when zero balloons were collected — there is no adjusted score to report, and the all-balloon mean is a different estimator (see avg_pumps_all_balloons). |
| `explosion_rate` | Fraction of balloons that exploded (0.0 – 1.0) |
| `mean_latency_between_pumps` | Average inter-pump interval in ms (computed via NumPy) |
| `total_balloons` | Total number of balloons in the session |
| `total_pumps` | Total pump events across all balloons |
| `total_explosions` | Number of balloons that exploded |
| `total_collections` | Number of balloons successfully collected |
| `learning_rate` | Rate of behavioral adaptation across trials (-1 to 1, higher = faster learning) |
| `risk_adjustment_score` | Ability to adjust behavior based on color cues (0-100) |
| `color_discrimination_index` | DEPRECATED: Use ev_efficiency_uniformity instead. Kept for backward compatibility. |
| `rng_normalized_pumps` | Mean pumps as ratio of EV-optimal stop per color, averaged across colors. 1.0 = pumping at exactly EV-optimal. >1.0 = over-pumping. <1.0 = conservative. Uses collected (non-exploded) balloons only. |
| `orange_avg_pumps` | Mean pump count on collected orange (high-risk) balloons. None when orange has insufficient collected balloons. |
| `patience_index` | Low-risk balloon average pumps (purple balloons) |
| `response_consistency` | Coefficient of variation in pump latencies (lower = more consistent) |
| `adaptive_strategy_score` | Composite score (0-100). Fixed weights: calibration 35%, learning 25%, uniformity 25%, money efficiency 15%. Exploratory: an unnormed, arbitrarily-weighted index — not a validated dependent variable; do not report without independent validation. |
| `risk_sensitivity` | Correlation between risk level and pumping behavior (-1 to 1) |
| `impulsivity_index` | Latency-based impulsivity index [0-1]. Derived from mean inter-pump latency: 1 - clamp(latency / 800ms, 0, 1). Higher = faster/more reflexive pumping. Exploratory: a derived latency metric with no source in the Lejuez canon — Lejuez et al. (2002) records no latency measure (impulsivity there is a self-report scale correlated with adjusted pumps); treat as descriptive, not a validated impulsivity measure. |
| `avg_pumps_all_balloons` | Mean pumps across ALL balloons regardless of outcome. Unlike average_pumps_adjusted (collected only), this includes exploded balloons and is not subject to censoring bias from RNG explosion-point selection. |
| `patience_index_normalized` | Purple (low-risk) EV-efficiency: EV(participant_purple) / EV(optimal_purple). Peaks at EV-optimal play (11 pumps) and decreases with both under- and over-pumping. Distinguishes patience from reckless over-pumping. |
| `half_split_learning_rate` | Learning rate from first-half vs second-half trial comparison per color. More robust than regression-based learning_rate at N=10 per color because no single outlier trial can dominate. Positive = improved performance in second half. Range: -1 to 1. |
| `within_balloon_consistency` | Mean CV of intra-pump latencies WITHIN individual balloons. Measures timing consistency during a single inflation sequence. Unlike response_consistency, immune to between-balloon strategy shifts. |
| `between_balloon_consistency` | CV of pump counts across all balloons. Measures how variable the participant's pumping strategy is balloon-to-balloon. High = erratic strategy; low = consistent strategy. |
| `ev_ratio_score` | EV-Ratio Risk Calibration: EV(participant) / EV(optimal) × 100 per color, weighted average. Derived from sequential model. 100 = perfectly optimal. |
| `explosion_penalty` | Excess explosion rate vs expected-at-optimal, averaged across colors. 0 = no excess explosions; 1 = maximum excess. Penalizes over-pumping. |
| `risk_calibration_score` | EV-Ratio Risk Calibration: EV(participant) / EV(optimal) × 100, EV-weighted across colors. 100 = perfectly EV-optimal pumping. Explosion penalty is reported separately to avoid double-penalizing. |
| `ev_efficiency_uniformity` | 1 - CV(per_color_ev_efficiencies). Measures how uniform efficiency is across risk levels. High = consistent efficiency (but not necessarily high). None if fewer than 2 colors have sufficient collected data. |
| `money_collected` | Total money earned from collected balloons (pumps × $0.25). |
| `money_efficiency` | Money collected / the study's expected earnings under EV-optimal play (sum of trials x EV-optimal per color, derived from the config's curves). 1.0 = earned what an EV-optimal session expects; >1.0 = better than expected (e.g. lucky survival past the optimum). Clipped to [0, 2]. |
| `flat_strategy_detected` | True if participant appears to use undifferentiated pumping strategy (same target across all colors). |
| `color_discrimination_trajectory` | Change in purple-vs-orange discrimination from first to last session third, normalized by EV-optimal spread (≈9 pumps). +1 = gained one optimal-spread of differentiation. 0 = stable. Negative = converged (reduced over-spread). None if insufficient color data. |
| `post_explosion_sensitivity` | Mean pump change on the next same-color balloon after an explosion, normalized by EV-optimal stop for the color. Positive = participant reduced pumps after pops (adaptive). Range approx [-2, 2]. None if no same-color follow-ups after explosions. |
| `tercile_learning_rate` | Learning rate from first-third vs last-third trials per color. Drops the noisy middle third to capture late learners more sharply. Same directional logic as half-split. Range [-1, 1]. |
| `session_valid` | True if the session passes all validity checks (complete, chronological, non-anomalous). |
| `qc_fast_response_trials` | Number of trials containing at least one inter-pump gap faster than the study's fast-response threshold. |
| `qc_zero_pump_streak` | Longest run of consecutive trials with zero pumps. |
| `qc_flagged` | True when any QC rule tripped (annotate-only; nothing is excluded). |
| `qc_fast_response_ms` | The fast-response threshold (ms) this session was judged against — recorded so a flag's criteria can be stated post hoc. |
| `qc_zero_pump_streak_threshold` | The zero-pump streak length this session was judged against — recorded so a flag's criteria can be stated post hoc. |
| `purple_average_pumps` | Mean pumps for this color across ALL balloons (both exploded and collected) |
| `purple_behavioral_avg_pumps` | Mean pumps using collected-only balloons (behavioral intention). Falls back to all balloons when fewer than 2 collected. |
| `purple_explosion_rate` | Explosion rate specific to this color |
| `purple_total_balloons` | Number of balloons of this color |
| `purple_collected_count` | Number of non-exploded (collected) balloons of this color |
| `purple_risk_profile` | Risk classification (low/medium/high) |
| `purple_used_fallback` | True if behavioral_avg_pumps fell back to all balloons due to insufficient collected count |
| `purple_ev_efficiency` | EV(behavioral_avg) / EV(optimal) for this color. None if insufficient collected data. |
| `purple_ev_optimal_stop` | Dynamically computed EV-optimal pump count for this color. |
| `purple_excess_explosion_rate` | Observed explosion rate minus expected rate at EV-optimal. Positive = over-pumping. |
| `teal_average_pumps` | Mean pumps for this color across ALL balloons (both exploded and collected) |
| `teal_behavioral_avg_pumps` | Mean pumps using collected-only balloons (behavioral intention). Falls back to all balloons when fewer than 2 collected. |
| `teal_explosion_rate` | Explosion rate specific to this color |
| `teal_total_balloons` | Number of balloons of this color |
| `teal_collected_count` | Number of non-exploded (collected) balloons of this color |
| `teal_risk_profile` | Risk classification (low/medium/high) |
| `teal_used_fallback` | True if behavioral_avg_pumps fell back to all balloons due to insufficient collected count |
| `teal_ev_efficiency` | EV(behavioral_avg) / EV(optimal) for this color. None if insufficient collected data. |
| `teal_ev_optimal_stop` | Dynamically computed EV-optimal pump count for this color. |
| `teal_excess_explosion_rate` | Observed explosion rate minus expected rate at EV-optimal. Positive = over-pumping. |
| `orange_average_pumps` | Mean pumps for this color across ALL balloons (both exploded and collected) |
| `orange_behavioral_avg_pumps` | Mean pumps using collected-only balloons (behavioral intention). Falls back to all balloons when fewer than 2 collected. |
| `orange_explosion_rate` | Explosion rate specific to this color |
| `orange_total_balloons` | Number of balloons of this color |
| `orange_collected_count` | Number of non-exploded (collected) balloons of this color |
| `orange_risk_profile` | Risk classification (low/medium/high) |
| `orange_used_fallback` | True if behavioral_avg_pumps fell back to all balloons due to insufficient collected count |
| `orange_ev_efficiency` | EV(behavioral_avg) / EV(optimal) for this color. None if insufficient collected data. |
| `orange_ev_optimal_stop` | Dynamically computed EV-optimal pump count for this color. |
| `orange_excess_explosion_rate` | Observed explosion rate minus expected rate at EV-optimal. Positive = over-pumping. |

## Trials CSV columns (`hazard-suite_trials.csv`)

One row per trial (balloon), in long format.

| Column | Description |
|--------|-------------|
| `timestamp_utc` | Session write time (UTC, filename-safe format) — shared by the session's files and its study-wide CSV rows. |
| `session_id` | The client-generated session identifier (UUID). |
| `candidate_id` | The participant ID entered at the start of the run. |
| `trial` | 1-based trial index within the session |
| `balloon_color` | the balloon's color name |
| `hazard_family` | the hazard family this balloon's color ran under (from the study config); None when the color is not in the config |
| `pumps` | pumps on this balloon |
| `outcome` | 'collected' (banked) or 'exploded' (popped) |
| `trial_earnings` | pumps × reward_per_pump when collected; 0 when popped |
| `mean_latency_between_pumps` | mean gap between this trial's successive pumps (ms); None when the trial has fewer than two pumps |

## Per-session files

Each completed session also writes four files of its own, namespaced by study title, candidate ID, and a UTC timestamp.

### Raw event telemetry (`*_events.jsonl`)

One JSON object per game event:

| Field | Description |
|--------|-------------|
| `timestamp` | Monotonic timestamp in ms (via performance.now()) |
| `type` | Event type identifier (e.g. 'pump', 'explode', 'collect') |
| `payload` | Event-specific data |

`payload` fields:

| Field | Description |
|--------|-------------|
| `balloon_id` | BART: which balloon |
| `color` | BART: balloon color (purple/teal/orange) for multi-risk profiles |
| `balloon_color` | BART: alternative field name for balloon color |
| `stimulus` | Stimulus value |
| `response` | Player response |
| `position` | Grid/sequence position |

### Scored metrics (`*_metrics.json`)

The full scored output: every Master CSV metric column above, un-flattened, plus:

| Field | Description |
|--------|-------------|
| `color_metrics` | Per-color performance metrics for purple/teal/orange balloons |
| `behavioral_profile` | Narrative behavioral insights (risk style, adaptability, etc.). Exploratory: risk_style is a hand-tuned heuristic decision tree, not a validated or normed classifier — treat as a descriptive summary, not a dependent variable. |
| `ev_optimal_stops` | Dynamically computed EV-optimal stopping points per color, plus per-color EV-efficiency values. |
| `session_warnings` | Validation warnings for this session (empty if fully valid). |
| `payout_amount` | Amount actually owed: money_collected × payout.rate, rounded half-up to 2 decimals. None when the study has no payout block. |
| `payout_currency` | The payout block's freeform currency label ('₺', '$', 'credits'). |

Each `color_metrics` entry:

| Field | Description |
|--------|-------------|
| `color` | Balloon color (purple/teal/orange) |
| `average_pumps` | Mean pumps for this color across ALL balloons (both exploded and collected) |
| `behavioral_avg_pumps` | Mean pumps using collected-only balloons (behavioral intention). Falls back to all balloons when fewer than 2 collected. |
| `explosion_rate` | Explosion rate specific to this color |
| `total_balloons` | Number of balloons of this color |
| `collected_count` | Number of non-exploded (collected) balloons of this color |
| `risk_profile` | Risk classification (low/medium/high) |
| `used_fallback` | True if behavioral_avg_pumps fell back to all balloons due to insufficient collected count |
| `ev_efficiency` | EV(behavioral_avg) / EV(optimal) for this color. None if insufficient collected data. |
| `ev_optimal_stop` | Dynamically computed EV-optimal pump count for this color. |
| `excess_explosion_rate` | Observed explosion rate minus expected rate at EV-optimal. Positive = over-pumping. |

### Session envelope (`*_session.json`)

The session's identity, design assignment, and per-session provenance (the raw telemetry stays in `*_events.jsonl`):

| Field | Description |
|--------|-------------|
| `session_id` | Unique session identifier (UUID string) |
| `game_type` | Which cognitive assessment game was played |
| `candidate_id` | Identifier for the candidate being assessed |
| `condition` | assigned condition for between-subject designs; one of the study config's declared `conditions` (issue 37), None when the study has none |
| `duplicate_acknowledged` | True when the ID screen warned that this candidate_id already had recorded sessions and the researcher chose to continue (issue 38) — keeps accidental ID reuse visible in the data |
| `practice` | True for Test Run sessions (issue 43): written under the practice/ subfolder of the output directory and never appended to the study-wide CSVs — inspectable, but never mingled with official data |
| `station_id` | the label of the station (machine) that recorded this session (DATA-SPEC §2.3); None on single-station data, where no station identity is configured |
| `id_source` | whether candidate_id was auto-generated by the ID screen's Generate button or hand-typed (DATA-SPEC §3.2); None on data recorded before — or without — the auto_participant_id option |
| `engine` | server-authored engine provenance (DATA-SPEC §3.1), stamped at write time; None on data recorded before the stamp existed — such sessions are ungraded at the Hub, never rejected |

`engine` fields (stamped by the instrument at write time):

| Field | Description |
|--------|-------------|
| `app_version` | The instrument (app) version that recorded this session. Currently identical to engine_version (released in lockstep); recorded separately so a future app/engine split stays representable. |
| `engine_version` | The scoring-engine version that ran this session — drives the Hub's verify-grading of stored metrics against re-scored ones. |
| `platform` | The OS/platform string of the machine that ran this session — per-station attribution that the last-writer-wins study-level provenance record cannot preserve. |

### Study config snapshot (`*_config.json`)

The exact study configuration this session ran under — the same shape as the frozen `hazard-suite_study.json`:

| Field | Description |
|--------|-------------|
| `schema_version` | data format version of study.json. 1.1 adds the optional standalone and metrics_mode fields (and auto_participant_id, landing with the auto-ID issue under this same bump); the bump is documentary, not load-bearing — pydantic defaults validate a 1.0 file with zero migration |
| `title` | the study's display title; slugged into every output filename |
| `language` | participant-facing language |
| `reward_per_pump` | currency units per banked pump |
| `currency` | Freeform label/symbol for the task-earnings units (reward_per_pump), shown with in-task money and the debrief earnings. Distinct from a payout block's currency, which labels the converted payout amount. |
| `seed` | RNG seed for the client burst sequence (SPEC §7.2). The webview mixes it with the participant ID (issue 61), so a fixed seed reproduces each participant from (seed, id) while participants diverge; null means a fresh run each time. Read only by the task webview, not the scorer. |
| `output_dir` | local folder for session data |
| `colors` | the study's color profiles, in display order |
| `conditions` | allowed condition names for between-subject designs; empty means this study has no conditions (issue 37) |
| `qc` | data-quality flag thresholds (issue 40); absent means the literature-informed defaults |
| `payout` | real-world payout conversion (issue 41); absent means no payout anywhere — the v1.0.0 behavior |
| `exit_passcode` | optional in-app kiosk lock (issue 44): while a session runs, every exit path asks for this passcode. Stored in study.json as deterrence, not security — it stops a curious participant, not an attacker with the preset file. Absent means exits are ungated — the v1.0.0 behavior |
| `standalone` | multi-station deployment mode: when true, stations write only per-session files — no live master-CSV appends — and the Hub rebuilds study-level outputs. Deliberately a config field, not a per-machine app setting, so every station of a study agrees by construction. Default off is the v1.0.0 single-station behavior |
| `metrics_mode` | reported metrics surface: the engine always computes the full advanced metrics; 'classic' projects every output (master CSV, trials CSV, metrics.json, data dictionary) down to the classic BART canon. Default 'advanced' is the v1.0.0 behavior |
| `auto_participant_id` | offer a Generate button on the ID screen that fills in a random 9-digit participant ID (DATA-SPEC §3.2) — an opt-in poka-yoke against cross-station ID collisions, and against the identical sequences two same-ID participants would otherwise see under a fixed seed. The field stays editable and manual entry stays freeform: the ID remains the researcher's external join key, never a mandatory random one. Deliberately independent of standalone. Default off is the v1.0.0 behavior |

Each `colors` entry:

| Field | Description |
|--------|-------------|
| `name` | color key used in metrics and CSV column names |
| `label` | participant-facing display name |
| `display_hex` | balloon fill color (hex) |
| `max_pumps` | N: hard cap on pumps for this color |
| `trials` | number of balloons of this color |
| `hazard` | the hazard family and parameters this color runs under |

Hazard-family parameters (`colors[].hazard`) are documented in the instrument's hazard-families reference.
